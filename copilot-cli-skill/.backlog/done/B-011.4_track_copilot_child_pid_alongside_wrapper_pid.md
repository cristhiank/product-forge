## Goal

Write actual copilot process PID to copilot.pid so lifecycle detection tracks the real process.

## Problem
worker.pid stores the Node.js wrapper PID. Wrapper spawns copilot as child. Wrapper can exit while copilot continues (or vice versa). Causes completed_no_exit status. Orchestrators resort to ps aux grep copilot.

## Implementation
1. worker-wrapper.js: after spawning copilot child, write child.pid to copilot.pid
2. workers.ts getStatus()/listWorkers(): check BOTH PIDs — running if either alive
3. workers.ts cleanup(): terminate both PIDs (wrapper first, then copilot)

## Files
- scripts/worker-wrapper.js, src/workers.ts, src/types.ts, tests/workers.test.ts

## Acceptance Criteria
- copilot.pid written by wrapper after spawn
- getStatus() reports running if either PID alive
- cleanup() terminates both PIDs
- No more completed_no_exit for workers still running
