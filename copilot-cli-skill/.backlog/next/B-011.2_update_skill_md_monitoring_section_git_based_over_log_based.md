## Goal

Update Monitoring Workers section in SKILL.md to recommend git-based monitoring (git log, git diff) as primary, demoting output.log to secondary.

## Problem
SKILL.md lines 269-341 recommend tail/grep on output.log. But output.log is heavily buffered by Copilot CLI — file size stays static for minutes. Orchestrators in 3+ sessions independently discovered git log/diff is far more reliable. This learned knowledge is lost between sessions.

## Changes
1. SDK vs Shell Decision Matrix: add "Check actual progress" row with git -C <worktree> log/diff
2. Shell Monitoring Patterns: add "Git-based progress check (Preferred)" BEFORE log-based sections
3. When to Use Each: update "What is it doing?" to recommend git diff --stat (primary), tail output.log (secondary)
4. Add buffering caveat to all output.log references
5. Add validateWorker() reference (after B-011.1 lands)
