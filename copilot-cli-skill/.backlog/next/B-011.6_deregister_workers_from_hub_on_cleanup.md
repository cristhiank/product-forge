## Goal

When sdk.cleanupWorker() is called, also deregister the worker from agents-hub so it does not persist as active forever.

## Problem
Workers registered in hub persist after cleanup. Hub DB retains them as active forever. Causes 20+ stale workers visible at session start.

## Options
1. copilot-cli-skill cleanup calls hub deregister API (couples the two skills)
2. agents-hub adds a reconcile/prune command that checks PID liveness (cleaner separation)
3. Both: cleanup emits an event, hub listens (most decoupled)

## Considerations
- Best-effort: do not fail cleanup if hub is unavailable
- The simpler fix may be on the agents-hub side: add hub worker prune
