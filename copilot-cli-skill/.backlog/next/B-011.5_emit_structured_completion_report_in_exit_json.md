## Goal

Enhance worker-wrapper.js to capture structured completion report at exit time, writing git info into exit.json.

## Problem
exit.json only contains exitCode and completedAt. Orchestrator must manually run git log, git diff, and build commands to understand what the worker did. Repetitive and wastes tokens.

## Implementation
In worker-wrapper.js finalize(), before writing exit.json, gather:
- commits: git log --oneline upstream..HEAD
- filesChanged: git diff --name-only upstream..HEAD
- hasDirtyWorkingTree: git status --porcelain

In workers.ts getStatus(), parse the enhanced exit.json and surface commits/filesChanged in WorkerStatus.

## Files
- scripts/worker-wrapper.js, src/workers.ts, src/types.ts

## Acceptance Criteria
- exit.json includes commit messages and files changed
- getStatus() surfaces this data without extra git commands
- Orchestrator can inspect worker output with single sdk.checkWorker() call
