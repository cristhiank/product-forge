## Goal

Update SKILL.md First Actions so stale dead workers are auto-cleaned at session start. Optionally add listAll({ autoCleanup: true }).

## Problem
At B-030 session start, 20 stale dead workers from prior sessions needed manual cleanup. Hub showed them as "active" even though PIDs were dead. Every new session begins with cleanup overhead.

## Changes
1. SKILL.md First Actions: move cleanupAll() to step 1 (before listing)
2. Optionally: add autoCleanup option to listAll() that removes non-running workers before returning the list, combining steps 1+2

## Files
- SKILL.md — First Actions section
- Optionally: src/workers.ts, src/sdk.ts, src/cli.ts

## Acceptance Criteria
- SKILL.md First Actions leads with cleanup before listing
