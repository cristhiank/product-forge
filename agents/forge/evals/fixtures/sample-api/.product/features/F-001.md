# F-001: Booking Management

## Job to be Done
When a pet owner calls to board their dog, the receptionist wants to quickly create a booking, so they can confirm availability and price.

## Problem Statement
Currently bookings are tracked in a spreadsheet. No validation, easy to double-book, revenue tracking is manual.

## Proposed Solution
CRUD API + dashboard for booking lifecycle (create, view, cancel). Real-time stats.

## Success Metrics
- Booking creation < 30 seconds
- Zero double-bookings
- Revenue dashboard accuracy 100%

## Out of Scope
- Online self-service booking (future F-003)
- Payment processing
- Multi-facility support
